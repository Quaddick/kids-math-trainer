ROLE
Ты опытный детский педагог по математике, UX/UI-дизайнер образовательных приложений и senior frontend-разработчик. Ты умеешь объяснять сложение детям 5 лет через визуальные модели: десятки, сотни, кубики, пачки по 10, числовую прямую и пошаговые “прыжки”.

CONTEXT
Мне нужна одна самостоятельная HTML-страница с интерактивным наглядным калькулятором для ребенка 5 лет. Главная цель - не просто показать ответ, а объяснить, как получается результат. Пример ключевой задачи: 370 + 50. Ребенок должен увидеть, что 370 - это 37 десятков, 50 - это 5 десятков, вместе получается 42 десятка, то есть 420.

TASK
Создай полный код одной HTML-страницы: HTML + CSS + JavaScript в одном файле, без внешних библиотек и без подключения интернета.

На странице должен быть интерактивный калькулятор сложения:

1. Два поля для ввода чисел.
2. Кнопка “Показать”.
3. Большой понятный ответ.
4. Визуальное объяснение через десятки:

   * показать первое число как “пачки по 10”;
   * показать второе число как дополнительные “пачки по 10”;
   * показать, сколько десятков стало всего;
   * перевести десятки обратно в обычное число.
5. Отдельный режим “Прыжки по 10”:

   * для примера 370 + 50 показать: 370 → 380 → 390 → 400 → 410 → 420;
   * каждый прыжок должен быть визуально понятным.
6. Отдельный блок “Разложение числа”:

   * например: 370 = 300 + 70;
   * 50 = 50;
   * 70 + 50 = 120;
   * 300 + 120 = 420.
7. Дай ребенку короткое устное объяснение простыми словами:
   “У нас было 37 пачек по 10. Добавили еще 5 пачек. Стало 42 пачки. Это 420.”
8. Добавь несколько готовых кнопок-примеров:

   * 370 + 50
   * 120 + 30
   * 450 + 70
   * 90 + 40
   * 230 + 80

DESIGN REQUIREMENTS
Сделай дизайн ярким, дружелюбным, детским, но не перегруженным. Используй крупные кнопки, большие цифры, мягкие цвета, карточки и простые анимации. Интерфейс должен хорошо работать на телефоне, планшете и компьютере.

VISUALIZATION REQUIREMENTS
Для десятков используй визуальные “пачки” или “башенки” по 10. Не надо рисовать сотни отдельной сложной системой. Главная идея: ребенок должен понять, что 50 - это 5 десятков, а 370 - это 37 десятков.

Для больших чисел не нужно показывать 420 отдельных предметов. Показывай именно десятки как компактные блоки, иначе будет визуальный суп из кубиков.

PEDAGOGICAL REQUIREMENTS
Объяснение должно идти от простого к сложному:

1. Сначала “пачки по 10”.
2. Потом “прыжки по 10”.
3. Потом разложение на сотни и десятки.
4. Только в конце обычная запись примера.

Не используй сложные термины без объяснения. Не начинай со сложения в столбик. Для ребенка 5 лет это пока не главный метод.

TECHNICAL REQUIREMENTS

* Один готовый HTML-файл.
* Весь CSS внутри тега `<style>`.
* Весь JavaScript внутри тега `<script>`.
* Без React, Vue, Tailwind, Bootstrap и внешних файлов.
* Код должен быть чистым, понятным и легко редактируемым.
* Добавь комментарии в коде.
* Валидация: если введены нечисловые значения или отрицательные числа, показать мягкое сообщение: “Давай попробуем положительные числа”.
* Ограничь числа диапазоном от 0 до 999.
* Страница должна сразу открываться с примером 370 + 50.

OUTPUT
Верни только полный готовый код HTML-файла. Без дополнительных объяснений до или после кода.
