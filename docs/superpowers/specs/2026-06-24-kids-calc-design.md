# Детский математический тренажёр (сложение + величины) — Дизайн

**Дата:** 2026-06-24
**Статус:** Утверждён к написанию плана
**Источник идеи:** [brief.md](../../../brief.md)

## 1. Цель

Интерактивное обучающее веб-приложение для ребёнка, помогающее **понять числа**.
Состоит из двух разделов, между которыми переключаются вкладками:

1. **Сложение** — наглядный калькулятор сложения. Главная цель не «показать
   ответ», а **объяснить, как он получается**: через блоки по разрядам, прыжки по
   10 и разложение числа. Ключевой пример `370 + 50`: ребёнок видит, что 370 — это
   37 десятков, 50 — 5 десятков, вместе 42 десятка, то есть 420.
2. **Соотношение величин** — наглядное сравнение порядков чисел (тысяча → … →
   триллиард). Отвечает на детские вопросы вида «миллион больше триллиона?»,
   «200000 больше миллиарда?» через лесенку порядков и режим сравнения двух чисел.

## 2. Аудитория и ключевой принцип

- Ребёнок **узнаёт цифры, но ещё не читает буквы**.
- Следствие (главный принцип дизайна): **смысл несут числа, цвет, форма и
  движение**, а текстовые подписи дублируют это для взрослого рядом. Страница
  должна быть понятна ребёнку без чтения текста.
- Озвучки нет (сознательное решение — конфликтует с ограничением «один офлайн-файл
  без внешних ресурсов»). Архитектура не закладывает аудио.

## 3. Объём первой версии

- **Два раздела** с навигацией вкладками: «Сложение» и «Соотношение величин».
- Раздел «Сложение»: **только сложение** (вычитание — возможное будущее
  расширение, в этой версии не делаем), **три уровня сложности** (переключатель),
  страница «растёт» вместе с ребёнком.
- Раздел «Соотношение величин»: лесенка порядков (тысяча → триллиард, шаг ×1000) +
  режим сравнения двух чисел.
- Без озвучки.
- **Vite + модульный vanilla JS/CSS** (без UI- и CSS-фреймворков), собирается в
  статический сайт. **PWA**: устанавливается на домашний экран iPad,
  полноэкранный режим, офлайн через service worker.
- **Деплой:** GitHub Pages и/или собственный VPS (оба дают HTTPS, необходимый для
  PWA/service worker).

## 4. Уровни сложности

Переключатель уровней вверху страницы. Уровень задаёт диапазон ввода, набор
кнопок-примеров и вид блоков визуализации.

| Уровень | Диапазон ввода | Визуальная модель | Примеры |
|---|---|---|---|
| 🐤 До 20 | 0–20 | Рамка-десятка (ten-frame): кубики в ячейках 2×5 | 8 + 5, 7 + 6, 9 + 4 |
| 🐰 До 100 | 0–100 | Палочки-десятки + кубики-единицы | 30 + 40, 23 + 14, 50 + 35 |
| 🦉 До 999 | 0–999 | Квадраты-сотни + палочки-десятки + кубики | 370 + 50, 120 + 30, 450 + 70, 90 + 40, 230 + 80 |

Это решает противоречие «5 лет vs числа до 999»: малыш начинает с «до 20», тот же
интерфейс ведёт его к сотням.

**Старт по умолчанию:** уровень «До 999», пример `370 + 50`, показанный сразу при
открытии (как требует бриф).

## 5. Структура страницы

```
┌─────────────────────────────────────┐
│  Заголовок + переключатель уровней    │
├─────────────────────────────────────┤
│  Ввод: [ 370 ] + [ 50 ] [ Показать ] │   крупные поля, экранные цифры
│  Кнопки-примеры: 370+50  120+30 ...  │
├─────────────────────────────────────┤
│  БОЛЬШОЙ ОТВЕТ: 420                   │
├─────────────────────────────────────┤
│  ① Блоки по разрядам   (анимация)     │
│  ② Прыжки по 10        (анимация)     │
│  ③ Разложение числа                   │
│  ④ Обычная запись: 370 + 50 = 420     │
├─────────────────────────────────────┤
│  Подсказка-объяснение простыми слова  │
└─────────────────────────────────────┘
```

Четыре обучающих блока — карточки-«шаги» сверху вниз. По умолчанию раскрываются
**по очереди** с кнопкой «Дальше →» (чтобы ребёнок не терялся); есть возможность
показать все сразу.

## 6. Обучающий поток (от простого к сложному)

Порядок строго педагогический, как в брифе:

### ① Блоки по разрядам — «сколько стало всего»
- Первое число выкладывается своими блоками, второе — другим оттенком тех же
  блоков.
- Ключевая анимация переноса разряда: **10 единиц «склеиваются» в 1 палочку-
  десяток; 10 палочек → в 1 квадрат-сотню**.
- Цвет кодирует разряд (см. §8). Для больших чисел показываем именно разрядные
  блоки (десятки/сотни), а не сотни отдельных предметов — никакого «визуального
  супа».

### ② Прыжки по 10 — числовая прямая
- Числовая прямая от первого числа.
- **Маскот-Львёнок 🦁** скачет дугами по +10: `370 → 380 → 390 → 400 → 410 → 420`.
- Каждый прыжок подписан числом.
- Если второе число не кратно 10 (например 23 + 14) — сначала прыжки по 10, затем
  один маленький до-прыжок на единицы.

### ③ Разложение числа
- `370 = 300 + 70`, `50 = 50`.
- Поразрядное сложение: `70 + 50 = 120`, затем `300 + 120 = 420`.
- Цвета разрядов совпадают с блоком ①.

### ④ Обычная запись
- Итоговая строка крупно: `370 + 50 = 420`. В самом конце — не начинаем с неё.

### Подсказка простыми словами
- Меняется под конкретный пример, например:
  «Было 37 десятков, добавили 5 — стало 42 десятка. Это 420.»

## 6a. Раздел «Соотношение величин»

Цель — наглядно показать **порядок** больших чисел и отвечать на вопросы «что
больше» и «во сколько раз». Масштаб нельзя показать «количеством предметов»
(10¹⁵ точек не нарисуешь), поэтому основа — лесенка порядков и сравнение.

### Навигация между разделами
- Вверху приложения — крупные touch-вкладки: **➕ Сложение** | **📊 Величины**.
- Уровни сложности (§4) относятся только к разделу «Сложение». Раздел «Величины»
  независим.

### Ряд величин (обиходный ряд ×1000)
- Точка отсчёта **1**, далее каждый порядок в 1000 раз больше предыдущего:

| Название | Степень | Число |
|---|---|---|
| тысяча | 10³ | 1 000 |
| миллион | 10⁶ | 1 000 000 |
| миллиард | 10⁹ | 1 000 000 000 |
| триллион | 10¹² | 1 000 000 000 000 |
| триллиард | 10¹⁵ | 1 000 000 000 000 000 |

- Примечание для взрослого (мелким текстом в интерфейсе/доке): строго научно
  10¹⁵ называется «квадриллион»; «триллиард» — обиходное название. Выбран
  обиходный ряд ×1000 ради ровной последовательности «каждый шаг ×1000».

### A. Лесенка порядков
- Вертикальная, снизу вверх: точка отсчёта **1**, затем тысяча … триллиард.
  **Выше = больше.**
- Каждая ступень — карточка: название, степень (для взрослого), полное число с
  разделителями групп по 3 (`1 000 000`), свой цвет порядка, количество нулей.
- Между ступенями — метка **×1000**.

### B. Режим сравнения двух чисел
- Два слота: **A** и **B**.
- Заполнение каждого слота: либо **тап по готовой величине** на лесенке, либо
  **ввод произвольного числа** экранной клавиатурой (та же `keypad`, что в
  калькуляторе), например `200 000`.
- Результат:
  - на лесенке подсвечиваются позиции A и B; произвольное число встаёт между
    соседними ступенями (например, 200 000 — между тысячей и миллионом);
  - крупный визуальный вердикт: знак **>** / **<** / **=** и стрелка к большему;
  - текст для взрослого («триллион больше миллиона»).
- **«Во сколько раз»:** для двух круглых величин ряда показываем точное кратное
  (×1000, ×1 000 000, …), так как шаг ровно ×1000. Для произвольных чисел —
  только позиция на лесенке и какое больше (без точного множителя).

### Примеры (вопросы ребёнка)
- «200000 больше миллиарда?» → 200 000 встаёт ниже миллиона, миллиард намного
  выше → вердикт **миллиард > 200 000**.
- «миллион больше триллиона?» → **триллион > миллион**, подпись «в 1 000 000 раз
  больше».

## 7. Данные и валидация

- Разбор числа на разряды: `сотни = ⌊n/100⌋`, `десятки = ⌊n/10⌋ % 10`,
  `единицы = n % 10`.
- Ввод ограничен диапазоном текущего уровня. Набор кнопок-примеров подобран так,
  что сумма не превышает ~999.
- Валидация:
  - нечисловое или отрицательное → мягкое сообщение «Давай попробуем
    положительные числа»;
  - выход за диапазон уровня → мягкое сообщение вида «Возьмём числа поменьше 🙂».
- Кнопка «Показать» при некорректном вводе не ломает страницу — показывает
  сообщение и не строит визуализацию.

**Раздел «Соотношение величин»:**
- Произвольное число для сравнения — целое от 0 до триллиарда (10¹⁵).
- Большие числа форматируются с разделителями групп по 3 (`200 000`,
  `1 000 000 000`).
- Пустой/некорректный слот сравнения — мягкая подсказка «Выбери два числа»,
  без поломки страницы.

## 8. Визуальный стиль

- Яркий, дружелюбный, детский, но не перегруженный.
- **Кодировка разрядов цветом:** сотни — фиолетовый, десятки — бирюзовый,
  единицы — оранжевый. Одинаково во всех блоках (①, ③) для связности.
- Мягкие тёплые цвета фона, светлая тема, крупная скруглённая типографика,
  карточки, простые анимации.
- Маскот — **Львёнок** (дружелюбный, узнаваемый, используется на числовой прямой
  и как «ведущий» персонаж).
- В разделе «Величины» каждый порядок (тысяча, миллион, …) имеет свой цвет
  ступени; палитра согласована с общим стилем приложения.

## 9. iPad / touch-first UX (требование заказчика)

Страница в первую очередь открывается в браузере на **iPad**; управление —
**пальцем**, не курсором.

- Все интерактивные элементы — крупные тач-цели (минимум ~48px, кнопки-цифры и
  «Показать» крупнее).
- **Никаких взаимодействий, завязанных только на hover**; визуальный отклик — на
  `:active`/нажатие.
- Числа вводятся через **собственную экранную цифровую клавиатуру** на странице
  (крупные кнопки 0–9, «стереть», «очистить»), чтобы не зависеть от системной
  клавиатуры; поля ввода — `readonly`/`inputmode="numeric"`, без всплытия мелкой
  системной клавиатуры.
- Корректный `<meta name="viewport">`; запрет случайного масштабирования
  двойным тапом и нежелательного выделения текста (`touch-action`,
  `user-select: none` на управляющих элементах).
- Достаточные отступы между кнопками, чтобы не промахиваться пальцем.
- Раскладка проверяется в портретной и альбомной ориентации iPad; адаптив также
  под телефон и ПК (CSS Grid/Flex, относительные единицы, медиазапросы).
- **PWA-режим:** `display: standalone`, иконки и `manifest.json` — после установки
  на домашний экран iPad открывается на весь экран без адресной строки Safari
  (ребёнок не уйдёт случайно на другую страницу).

## 10. Технические решения и архитектура

- **Стек:** Vite (Node 18+; в окружении Node 22) + vanilla JS (ES-модули) + CSS.
  Без React/Vue/Tailwind/Bootstrap. Vite — только инструмент разработки и сборки;
  в рантайме никаких UI-/CSS-фреймворков и сетевых запросов.
- **Модульная структура** (каждый модуль — одна ответственность, чистые функции
  рендера):
  - `levels.js` — конфигурация трёх уровней (диапазон, примеры, вид блоков);
  - `parse.js` — разбор числа на разряды, валидация;
  - `renderBlocks.js` — блоки по разрядам + анимация переноса;
  - `renderJumps.js` — числовая прямая и прыжки Львёнка;
  - `renderDecomposition.js` — разложение числа;
  - `renderPlain.js` — обычная запись примера;
  - `keypad.js` — экранная цифровая клавиатура (используется в обоих разделах);
  - `magnitudes.js` — ряд величин (значение, название, степень, цвет, число нулей);
  - `renderLadder.js` — лесенка порядков (раздел «Величины»);
  - `renderCompare.js` — сравнение двух чисел, вердикт, позиционирование на
    лесенке;
  - `router.js` — переключение между разделами «Сложение» и «Величины»;
  - `main.js` — состояние и связывание модулей;
  - `style.css` — стили; `index.html` — разметка-каркас.
- **PWA:** `manifest.json` (имя, иконки, `display: standalone`, тема) +
  service worker для офлайн-кэширования (через `vite-plugin-pwa` или ручной SW).
- Анимации — CSS transitions/keyframes + минимальный JS-таймер для пошаговости;
  уважать `prefers-reduced-motion`.
- Код чистый, легко редактируемый, **комментарии на русском**.
- На этапе реализации UI задействовать скилл **frontend-design** для качества
  вёрстки и визуального уровня.

## 10a. Разработка и деплой

- **Разработка:** `npm run dev` (Vite). Просмотр на iPad по локальной сети через
  `vite --host` (открыть `http://<IP-компа>:5173` в Safari, hot-reload).
- **Сборка:** `npm run build` → статический сайт в `dist/`.
- **Деплой:** GitHub Pages и/или собственный VPS. Оба обеспечивают HTTPS,
  необходимый для регистрации service worker и установки PWA на iPad.
- **Установка на iPad:** Safari → «Поделиться» → «На экран Домой» → полноэкранное
  офлайн-приложение с иконкой.

## 11. Критерии готовности

**Раздел «Сложение»:**
- Приложение открывается на разделе «Сложение» сразу с примером `370 + 50`
  (уровень «До 999») и показывает все 4 блока объяснения.
- Переключение уровней меняет диапазон, примеры и вид блоков.
- Кнопки-примеры подставляют числа и запускают объяснение.
- Анимация переноса разряда (10 единиц → десяток, 10 десятков → сотня) работает.
- Львёнок прыгает по числовой прямой по +10 (и до-прыжок на единицы при
  необходимости).
- Валидация показывает мягкие сообщения, не ломая страницу.

**Раздел «Соотношение величин»:**
- Вкладки переключают разделы; уровни сложности не влияют на «Величины».
- Лесенка показывает ряд `1 → тысяча → … → триллиард` с названиями, числами и
  метками ×1000; выше = больше.
- В режиме сравнения можно выбрать величины с лесенки и/или ввести произвольное
  число; вердикт показывает, какое больше (знак и стрелка), произвольное число
  встаёт между нужными ступенями.
- Для двух круглых величин ряда показывается точное кратное (×1000 и т.д.).

**Общее:**
- Управление удобно пальцем на iPad (крупные цели, экранная клавиатура, без
  hover-зависимостей), раскладка корректна на телефоне/планшете/ПК.
- `npm run build` собирает статический сайт; PWA устанавливается на домашний
  экран iPad и работает офлайн (полноэкранный режим).
- Проект деплоится на GitHub Pages / VPS по HTTPS.

## 12. Вне объёма (YAGNI)

- Вычитание, умножение и прочие операции.
- Озвучка/аудио.
- В разделе «Величины»: метафоры из жизни (миллион секунд ≈ 12 дней и т.п.) и
  вложенный зум ×1000 — возможные будущие улучшения, в первой версии не делаем.
- Сохранение прогресса, аккаунты, статистика.
- UI-/CSS-фреймворки в рантайме, шрифты и картинки по сети (Vite — только
  инструмент сборки; иконки PWA — локальные ассеты в проекте).
